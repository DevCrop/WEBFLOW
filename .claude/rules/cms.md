# CMS Collections — Intellectual Data
# Build collection schema BEFORE creating any page element that references it.

---

## Collections Overview

| Collection       | Slug              | Purpose                              | Pages that use it           |
|------------------|-------------------|--------------------------------------|-----------------------------|
| Insight          | `insights`        | Thought leadership / blog posts      | Company > Insight           |
| Newsroom         | `newsroom`        | Press releases, news announcements   | Company > Newsroom          |
| Release Notes    | `release-notes`   | Product/solution update log          | Solutions > 릴리스 노트     |
| Contact          | `contact-submissions` | Form submission storage          | Contact Us 폼 연결          |
| Newsletter       | `newsletter-subscribers` | Newsletter subscription DB  | Footer / 구독 폼            |

> DB migration required for: Insight, Newsroom, Contact, Newsletter (from existing site)

## 4. Contact Submissions Collection

**Slug:** `contact-submissions`
Webflow forms do not auto-write to CMS. Route via webhook → Make/Zapier/n8n → CMS API.
Do NOT store this by embedding Webflow API token in client-side code.

| Field Name     | Slug           | Type        | Required | Notes                              |
|----------------|----------------|-------------|----------|------------------------------------|
| Name           | `name`         | PlainText   | ✓        | Form input `name="name"`           |
| Email          | `email`        | Email       | ✓        | Form input `name="email"`          |
| Company        | `company`      | PlainText   |          | Form input `name="company"`        |
| Phone          | `phone`        | Phone       |          | Form input `name="phone"`          |
| Category       | `category`     | Option      |          | Enum: 제품문의 · 파트너십 · 기타   |
| Message        | `message`      | PlainText   | ✓        | Form input `name="message"`        |
| Submitted At   | `submitted-at` | Date        | ✓        | Auto-filled by automation          |
| Status         | `status`       | Option      |          | Enum: 신규 · 확인중 · 완료         |

> Form input `name` attributes must exactly match field slugs for automation 1:1 mapping.

## Creation Checklist (per collection)

1. Create collection with slug exactly as listed above
2. Add fields in the order listed — slugs must match exactly
3. Add 2–3 placeholder items so the admin UI is visibly populated
4. Build the page UI that references this collection
5. (Contact / Newsletter) Register form webhook → wire to external automation

Tool: `data_cms_tool` for creation, `bulk-cms-update` skill for populating items.
